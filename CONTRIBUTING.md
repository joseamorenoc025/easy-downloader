# Guía de Contribución

¡Gracias por tu interés en contribuir a **EasyDownloader**! Todas las contribuciones, reportes de errores y sugerencias son bienvenidas.

## ¿Cómo contribuir?

### 1. Reportar un Bug
Si encuentras un error, por favor asegúrate de que no haya sido reportado antes en los [Issues](../../issues). Si es nuevo, crea un Issue usando la plantilla de **Bug Report** y proporciona tanta información como sea posible (versión de Python, sistema operativo, pasos para reproducirlo).

### 2. Sugerir una Mejora
Si tienes una idea para mejorar la aplicación, abre un Issue usando la plantilla de **Feature Request**. Describe claramente qué problema resuelve tu sugerencia o por qué sería útil.

### 3. Contribuir con Código
Si quieres escribir código, sigue estos pasos:

1. **Haz un Fork** del repositorio a tu cuenta de GitHub.
2. **Clona** tu fork localmente:
   ```bash
   git clone https://github.com/tu-usuario/EasyDownloader.git
   ```
3. **Crea una rama nueva** para tu característica o corrección:
   ```bash
   git checkout -b feature/mi-nueva-caracteristica
   ```
   o
   ```bash
   git checkout -b fix/correccion-del-error
   ```
4. **Haz tus cambios** y asegúrate de que el código siga funcionando correctamente.
5. **Haz Commit** de tus cambios con mensajes claros y descriptivos:
   ```bash
   git commit -m "Añade soporte para descargas en 4K"
   ```
6. **Haz Push** de tu rama a tu repositorio en GitHub:
   ```bash
   git push origin feature/mi-nueva-caracteristica
   ```
7. Abre un **Pull Request (PR)** desde tu rama hacia la rama `main` de este repositorio. Describe claramente los cambios realizados en el PR.

## Estilo de Código
- Mantén el código limpio y legible.
- Usa nombres descriptivos para variables y funciones.
- Comenta las partes complejas del código.
- Asegúrate de no incluir claves personales o tokens en el código subido.

¡Gracias por ayudar a hacer EasyDownloader mejor!
